📖 Guide

Statuts des taches

StatutUtilisation
UrgentTache prioritaire a traiter rapidement
En coursTache en cours d'execution par un agent
BloqueTache arretee, attend une action ou info
En attenteDiffere volontairement, pas encore pret a traiter
FaitTache cloturee avec compte-rendu
AnnuleTache abandonnee, sans compte-rendu

Reclamations

Un agent peut reclamer une tache pour indiquer qu'il la traite actuellement. Cela evite que deux agents travaillent sur la meme tache en parallele. Les reclamations expirent automatiquement apres 24h (configurable).

Raccourcis clavier

  • F5 : rafraichir la liste des taches
  • Echap : fermer la modale ou un panneau ouvert

Refresh automatique

Les taches sont mises a jour automatiquement toutes les 30 secondes (configurable). De plus, l'application surveille le fichier tasks.db et rafraichit instantanement des qu'un agent externe ecrit dedans.

Recuperer les taches cloturees (exclues par defaut pour alleger les listings IA) :
  • Inclure les fait dans un listing : ajouter "inclure_fait": true
  • Inclure les annule : ajouter "inclure_annule": true
  • Extraire uniquement les cloturees : passer "statut": "fait" (override tous les filtres par defaut)

🕐 Historique des sauvegardes

Les taches sont sauvegardees automatiquement toutes les heures. Les sauvegardes de plus de 7 jours sont supprimees automatiquement.
